# Ants Reporting Policy

**Timestamp (JST):** 2026-02-10T15:05:00+09:00

## 1. 原則 (Principles)
1.  **日本語のみ (Japanese Only)**: 全ての報告は日本語で行う。英語の使用は禁止（コードブロック内の引用等は除く）。
2.  **最終報告のみ (Final Report Only)**: 作業途中の「進捗報告」は禁止。全ての作業と検証が完了した時点での「最終報告」1回のみを行う。

## 2. 運用ルール (Operational Rules)
### 2.1 報告プロセス
Antsがスレッドに報告を行う際は、以下の手順を必ず遵守する。

1.  **ファイル作成**: 報告内容は必ず事前に `docs/reports/` 配下にMarkdownファイルとして作成する。
    - ファイル名規則: `YYYY_MM_DD_report_name.md`
2.  **Preflight Check**: 作成したレポートファイルを含め、`scripts/ants_preflight.sh` を通過させる。
3.  **スレッド投稿**: Preflightを通過したファイルの内容に基づき、スレッドに「最終報告」として投稿する。

### 2.2 禁止事項
- `進捗報告` という見出しの使用。
- 英語での状況報告。
- 承認・マージ前の見切り報告。

## 3. 違反時の対応
- 本ポリシーに違反した場合（英語報告や途中経過報告）、その報告は即座に撤回（無効化）とし、正しい形式で再報告を行う。
- 再発防止のため、ツール側で物理的なゲート（Preflight check等）を強化する。
