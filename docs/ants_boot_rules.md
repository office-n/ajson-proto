# Ants起動ルール（日本語恒久版）

**作成日**: 2026-02-06  
**目的**: Ants起動時に常時日本語応答を恒久的に適用する

---

## 起動トリガー

ユーザーが「**起動**」と入力した際、このルールを読み込み適用する。

---

## 恒久ルール: 常時日本語応答

### 基本方針
- Antsの全応答は **日本語** で行う
- このルールは起動後、会話が続く限り有効（セッション終了まで継続）
- 例外: 技術的に英語が必須の場合のみ（詳細は下記）

### 日本語対象
- ✅ 通常の会話・説明・質問
- ✅ 証跡ファイル（evidence_*.md）
- ✅ ウォークスルー（walkthrough.md）
- ✅ タスクファイル（task.md）
- ✅ 実装計画（implementation_plan.md）
- ✅ PRタイトル・本文
- ✅ コミットメッセージ本文（タイトルは英語可）
- ✅ エラーメッセージの説明

### 日本語応答の特別ルール

#### タイムスタンプ表示（恒久ルール）

##### 全返信に必須
- **全ての返信の冒頭に必ず日本時間（Asia/Tokyo）でタイムスタンプを記載する**
- 形式（ISO 8601準拠）: `タイムスタンプ: YYYY-MM-DDTHH:MM:SS+09:00（Asia/Tokyo）`
- 例: `タイムスタンプ: 2026-02-06T19:38:00+09:00（Asia/Tokyo）`
- 適用箇所: 通常の会話、進捗報告、質問、全ての返信
- 目的: 全ての発言の時刻を明確に記録する

##### 「現在実施中」提示時
- **「現在実施中」を提示する場合、必ず日本時間でタイムスタンプを記載する**
- 形式: `### 現在実施中（2026-02-06T19:38:00+09:00）`
- 適用場面: 作業進捗報告、Phase進行状況、長時間タスクの中間報告
- 目的: ユーザーが「いつ時点の状況か」を明確に把握できるようにする

#### 完了予定時刻の計算・表示（恒久ルール）
- **「約X-Y分で完了予定」と記載する場合、完了予定時刻も計算して併記する**
- 計算方法: 現在のタイムスタンプ + X分 〜 現在のタイムスタンプ + Y分
- 形式: `約30-40分で完了予定（20:02-20:12 完了予定）`
- 24時間制で表示（HH:MM形式）
- 目的: ユーザーが具体的な完了時刻を把握できるようにする

**例**:
```
### 現在実施中（2026-02-06T19:32:00+09:00）

**🔄 Phase 3**: Console UX v2（大規模改修）
- Layout v2（Composer底固定、viewport基準高さ）実装開始
- 約30-40分で完了予定（20:02-20:12 完了予定）
```


### 例外（英語のまま）
- コード（変数名、関数名、クラス名等）
- コマンドライン（シェルコマンド、引数）
- ファイル名・ディレクトリ名
- URL
- JSONキー名、API仕様
- 技術用語で日本語訳が不自然な場合（例: Keychain、DRY_RUN）

## 起動時の宣言

起動トリガー「起動」を受けた際、必ず冒頭で以下を宣言する：

```
起動ルール読込完了。以後日本語で応答します。
```

この宣言により、ユーザーは日本語モードが有効化されたことを確認できる。

---

## 英語で応答してしまった場合の対処

もし誤って英語で応答してしまった場合：
1. 即座に日本語で同じ内容を再出力する
2. 以後は日本語を継続する
3. ユーザーに謝罪や説明は不要（淡々と日本語へ切り替える）

---

## 運用上の注意

### ユーザーからの入力言語
- ユーザーが英語で質問しても、Antsは日本語で応答する
- ただし、ユーザーが明示的に「英語で回答してください」と指示した場合は例外

### 技術文書の引用
- 公式ドキュメントやエラーメッセージを引用する際は原文（英語）のまま
- ただし、その前後の説明は日本語で行う

### コード内コメント
- 既存コードのコメントが英語の場合、新規追加分は日本語で書く
- 既存英語コメントの更新時は、文脈に応じて日本語化を検討

### 証跡ファイル（evidence）の規約
- **全ての evidence_*.md ファイルの先頭に必ずタイムスタンプ行を記載する**
- 形式（ISO 8601準拠）: `**タイムスタンプ**: YYYY-MM-DDTHH:MM:SS+09:00（Asia/Tokyo）`
- 例: `**タイムスタンプ**: 2026-02-06T19:38:00+09:00（Asia/Tokyo）`
- 禁止事項:
  - `file`+`://` 絶対パス混入禁止
  - APIキーなど機密情報の記載禁止
- 検証方法: `grep -RIn "file"":""//" evidence_*.md || true`（0件であること）

## 補足: なぜ日本語か

- ボス（ユーザー）の母語が日本語
- 技術的な内容も日本語で理解する方が早く正確
- 証跡やドキュメントを日本語で残すことで、チーム内の共有が容易
- Antsは多言語対応可能だが、明示的な指定がない限り日本語を優先
